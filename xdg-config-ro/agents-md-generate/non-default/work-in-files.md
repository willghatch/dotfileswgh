# Always write plans and results to files

The environment may disconnect; anything written only in chat will be lost.
Always write all plans, notes, reports, and results to your agent-work-directory per the File Organization instructions.
This includes final answers given in chat.
Any substantive response -- summary, analysis, report, or research result -- must be written to a file in your agent-work-directory before or alongside replying.
Reference the file path in your chat reply so the user can find it.

Before sending ANY reply that contains findings, a list, a summary, or a direct answer to a factual question, you MUST:
  1. Create your agent-work-directory if it does not yet exist.
  2. Write the full response content to a file inside it.
  3. Reference the file path in your chat reply.

There are NO exceptions for "simple" or "short" answers, requests for clarification, etc.
Do not even give a final answer in chat unless explicitly requested, merely tell where the key result file is.

## Task Lists

When working on a to-do list, you MUST write the to-do list to an org-mode file in your agent-work-directory.
The list should be very brief -- just TODO/IN-PROGRESS/DONE, and a task title in few words.
Where applicable, also add a git branch name for the task and/or a subagent agent-work-directory path.
You MUST update this list every time the status changes (IE a task is started or finished) so another agent can resume the task if it is interrupted.
